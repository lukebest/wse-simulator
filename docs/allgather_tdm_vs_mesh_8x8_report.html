1) 关键结论（分组交换 ND，slot=1）

  • k2_n6 (C=5) ND AllGather：全面优于 Mesh，TDM/Mesh 约为 0.39×~0.40×。
  • k4_n3 (C=9) ND AllGather：与 Mesh 互有胜负，TDM/Mesh 约为 0.99×~1.18×。
  • k8_n2 (C=16) ND AllGather：整体慢于 Mesh，TDM/Mesh 约为 2.99×~3.75×。
  • slot 1→4 对 k8_n2 的 ND makespan 比值约 2.30×~3.44×（color 数越多通常越敏感）。
  • 瓶颈分解：TDM 的 color_buffer_wait_cycles 与 link_wait_cycles 随 C 增大显著上升，是 k8_n2 落后的主因。
  • direct_allgather 在 64 节点下为 O(N²) 流量，TDM 全面慢于 Mesh；主评估应以 ND 分组交换为准。

5) 工程建议

  • 8×8 AllGather 若走 TDM FB，优先 k2_n6 + nd_dimension_exchange + slot=1（109~3267 cycles，相对 Mesh 约 0.39×~0.40×）。
  • k4_n3 小消息略优、中大消息略慢，可作为次优折中（1KB 0.99×，16KB 1.07×，256KB 1.18×）。
  • k8_n2 在当前 color/per-buffer 模型下不适合作为 AllGather 首选（C=16，ND 约为 Mesh 的 3×~3.75×）。
  • direct_allgather 仅作参考：TDM 因 color 等待与 O(N²) 流量叠加，makespan 显著高于 Mesh（例如 1KB 下 k8_n2 为 Mesh 的 2.98×）。
