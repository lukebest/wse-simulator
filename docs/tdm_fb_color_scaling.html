TDM Flattened Butterfly：Color 数 × Mesh 规模 × FB 规模

基于 wsesim 物理 max-load 扫描（= 贪心着色下界 C）。约束：kⁿ = S²（S×S mesh，N=S² PE）。

Color = 24 时，最大支持多大 Mesh？

32 × 32（1024 PE）

在全部合法 (k,n) 配置中，最大且 C≤24 的为 (k,n)=(2,10)：C=21，逻辑链路 10240 条，逻辑直径 10 跳。

注意：C=24 是上界；具体 (k,n) 需 C≤24 且满足 kⁿ=S²。不存在单一「mesh 规模→C」映射——同一 S 上不同 (k,n) 的 C 可差 4× 以上。

核心规律

C = max_{物理有向边 e} |{ 逻辑链路 ℓ : XY_route(ℓ) ∋ e }| = color_lower_bound
  • C 不等于逻辑维度 n、逻辑链路数 L、或 mesh 边总数。
  • C 等于最拥塞物理边上的逻辑路径条数（TDM 周期下界；已测配置贪心均达到）。
  • 约束：N = S² = kⁿ，L = N·n·(k−1)。
  • 同 N 下：k↑ n↓ → L↑ 且 C↑（长组内链路多）；k=2 n↑ → C 缓增，可撑更大 mesh。

经验公式

C = ⌊k²/4⌋
2-flat (n=2)，S×S mesh，k=S（实测 k=4…32）
C = (k−1)²
例：8×8 (4,3) → C=9
C ≤ 24 → k≤9
n=2 时：9×9 可行，10×10 超限 (C=25)
k=2 族
n=4→C=2, n=6→5, n=8→10, n=10→21

跨配置 Color 规律

  • k-ary, n=2（2-flat）：Color 类由 k 节点组内逻辑距离决定——C0=邻接，奇数距离拆为端点/中间两 Color，最大距离单独一 Color；C ≈ k（8×8 为 16）。
  • k=2（binary hypercube）：基础 2-Color 由坐标参考位奇偶决定；mesh 变大时跨维路径冲突增加，C 从 2 扩展至 5（8×8）或更高。
  • 4 节点组规则：4×4 (4,2) 与 8×8 (4,3) 共享同一距离—Color 映射；后者因 3 维叠加 + 更大 mesh，C 从 4 升至 9。
  • 单 Color 内：同一时隙所有链路物理边互不冲突；跨 Color 合并 = 完整 FB 逻辑拓扑。

设计含义（Cerebras 24 Color）

  • 若坚持 2-flat 高 radix（逻辑 2 跳、组内全连）：mesh 不宜超过 9×9。
  • 若可接受 binary 多 flat（逻辑 10 跳）：单程序可覆盖 32×32 = 1024 PE（C=21，余 3 个 color 余量）。
  • 8×8 (8,2) 用 C=16，余 8 个 color —— 与现有 4×4 (4,2) 用 C=4 占满下界 的比例类似。
  • 选 (k,n) 是在 逻辑直径、链路数 L、TDM 周期 C 之间的权衡，不能只看 mesh 边长。
