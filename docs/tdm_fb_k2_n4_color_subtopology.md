# 4×4 Mesh 上 2-ary, 4-flat Flattened Butterfly 的 Color 子拓扑分解

> **目标**：在 4×4 物理 2D Mesh 上，通过 **2 个 Color（时分信道）** 实现完整的
> 2-ary, 4-flat Flattened Butterfly 逻辑拓扑。每个 Color 承载一组**物理边不冲突**
> 的逻辑长链路，**2 个 Color 合并 = 完整 FB**。

---

## 1. 基本定义

### 1.1 节点编号（行优先）

```
 0   1   2   3        ← row 0
 4   5   6   7        ← row 1
 8   9  10  11        ← row 2
12  13  14  15        ← row 3
```

坐标 `(row, col)`: `node = row × 4 + col`

### 1.2 物理拓扑：4×4 2D Mesh

每个节点与上下左右邻居相连，共 24 条**无向**物理边（48 条**有向**物理边）。

### 1.3 逻辑拓扑：2-ary, 4-flat Flattened Butterfly

每个节点地址 `(d₃, d₂, d₁, d₀)`，其中每位 ∈ {0,1}，编码：

```
node_id = d₀ + 2·d₁ + 4·d₂ + 8·d₃
```

- **Dim 0**：翻转 d₀ → 每对节点 1 条有向链路（共 8 对 × 2 方向 = **16 有向**）
- **Dim 1**：翻转 d₁ → **16 有向**
- **Dim 2**：翻转 d₂ → **16 有向**
- **Dim 3**：翻转 d₃ → **16 有向**
- **合计**：**64 有向**逻辑链路

逻辑直径 = **4 跳**：任意两点最多经 4 个维度各 1 跳（维度序路由）。

与 (4,2) 对比：k=2 时每维只有 **2 节点对**（非 4 节点全连），链路数 64（非 96），TDM 周期 **C=2**（非 4）。

---

## 2. Color 分解方案

### 2.1 核心思路

k=2 时，每个维度上的逻辑链路都是 **2 节点翻转**（逻辑跳距恒为 1）。但映射到物理 mesh 后，
物理跳距为 **1 或 2**（取决于坐标位与 mesh 布局的对应关系），多条逻辑链路可能共享同一
有向物理边，因此仍需时分。

**关键观察**：物理边最大负载 = **2**，故至少需要 **2 个 Color**。下述方案恰好达到下界。

### 2.2 通用分配规则

对维度 `d ∈ {0,1,2,3}` 上的一条有向逻辑链路，设源节点坐标为 `(d₃,d₂,d₁,d₀)`，定义参考位：

```
j = (d+1) mod 4   当 d 为偶数 (0, 2)
j = (d-1) mod 4   当 d 为奇数 (1, 3)
```

| 维度 d | 参考位 j | Color 条件 |
|--------|---------|-----------|
| **0** (偶) | d₁ | **C0** 若 d₁=1，否则 **C1** |
| **1** (奇) | d₀ | **C0** 若 d₀=0，否则 **C1** |
| **2** (偶) | d₃ | **C0** 若 d₃=1，否则 **C1** |
| **3** (奇) | d₂ | **C0** 若 d₂=0，否则 **C1** |

**等价记忆**：偶数维看「高一位」坐标是否为 1；奇数维看「低一位」坐标是否为 0。

### 2.3 各维度链路分布

| 维度 | 物理跳距 | C0 链路数 | C1 链路数 | 特征 |
|------|---------|----------|----------|------|
| dim0 | 1 | 8 | 8 | 水平相邻对 (d₁=1→C0, d₁=0→C1) |
| dim1 | 2 | 8 | 8 | 跨 1 列 (d₀=0→C0, d₀=1→C1) |
| dim2 | 1 | 8 | 8 | 垂直相邻对 (d₃=1→C0, d₃=0→C1) |
| dim3 | 2 | 8 | 8 | 跨 1 行 (d₂=0→C0, d₂=1→C1) |
| **合计** | | **32** | **32** | |

### 2.4 为什么不冲突？

每个 Color 内 32 条链路使用的有向物理边**互不重叠**；同一物理边最多被 2 条逻辑链路
（不同 Color）共享——恰好填满负载下界。

## 4. 合并验证

| Color | 有向链路数 |
|-------|----------|
| C0 | 32 |
| C1 | 32 |
| **总计** | **64** = 2-ary, 4-flat FB 全集 ✓ |

物理边负载：每条有向物理边最多 **2** 条逻辑链路（分属 C0/C1），TDM 周期 **C=2** 最优。

---

## 5. 路由机制

### 5.1 Color 选择

对维度 `d` 上从 `src` 出发的一跳，按 §2.2 查表得 Color。多跳路由使用**维度序**（dim0→dim1→dim2→dim3），
每跳独立查 Color；若全程同色则单时隙完成。

### 5.2 路由示例

**示例 1**：`0 → 5`（(0,0,0,0)→(1,0,1,0)，差 d₀,d₂）

```
0 →(dim0,C1)→ 1 →(dim2,C1)→ 5     ← 单 Color C1
```

**示例 2**：`0 → 10`（差 d₁,d₃）

```
0 →(dim1,C0)→ 2 →(dim3,C0)→ 10    ← 单 Color C0
```

**示例 3**：`0 → 6`（差 d₁,d₂）

```
0 →(dim1,C0)→ 2 →(dim2,C1)→ 6    ← 跨 C0 + C1
```

**示例 4**：`0 → 15`（四位全差）

```
0 →(C1)→ 1 →(C1)→ 3 →(C1)→ 7 →(C1)→ 15    ← 单 Color C1（dim0→1→2→3 序）
```

**示例 5**：`1 → 14`（四位全差，但中间跳变色）

```
1 →(C1)→ 0 →(C0)→ 2 →(C1)→ 6 →(C1)→ 14    ← 跨 C0 + C1
```

### 5.3 单 Color 完成条件

当维度序路由每一跳的 Color 全部相同时，整条通信可在**一个时隙**内流水线完成。
k=2 时由于每维仅 1 位翻转，同色概率与坐标奇偶模式相关——并非所有 (src,dst) 都能单色完成。

---

## 6. 时分调度

```
时隙:   t=0      t=1      t=2      ...
Color:  C0       C1       C0       ...
```

跨 Color 路由额外等待 = 两跳 Color 编号之差（mod 2），最多 **1 个时隙**。

|------|-------------------|-------------------|

| 物理边最大负载 | 4 | **2** |
